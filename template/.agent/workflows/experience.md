# /experience , Extract Capability Timeline

> *"Experience is not a resume. It is a ledger of conquered complexity."*

EXPERIENCE is the second verb of the PERSON OS. It maps your historical timeline against your established identify to build a capability ledger.

---

## Usage

```
/experience
```

## Step 1: Pre-Reasoning Governance (PASS) -> HUMAN APPROVAL REQUIRED

Before starting, restate the user's intent back to them using the PASS framework. Apply these permanent directives to their specific query:

- **Purpose**: Map the user's historical capability against their baseline identity.
- **Anti-Purpose**: Do NOT allow the user to list chronological resumes or tasks devoid of complexity. Break chronologies.
- **Success-Signal**: We extract specific tensions overcome; output is a STRICT 5-column STORY ledger.
- **Synthesis**: Transforms tacit memory into an explicit ledger (`experience/[milestone].md`) that will fuel strategy and narrative.

Present the calculated PASS table to the user and ask: *"Do you approve this execution intent?"*
**STOP AND WAIT FOR EXPLICIT HUMAN APPROVAL.**

## Step 2: Context Injection
*(Execute ONLY after PASS approval)*

Before asking the user anything, you **MUST** read the baseline identity:
**Source**: `profile/core.md`

If this file does not exist, abort and tell the user to run `/profile` first.

## Step 3: Interrogate

Using their parsed `core.md`, ask the user to provide their career timeline.
Ask: *"Based on your profile as [Identity], what are the 3 most significant milestones where you proved your capability?"*

## Step 4: Synthesis Framework (STORY)

Synthesize their answers into an `Experience Ledger` using the **STORY** framework. Format it as an exact table:
- **S** - Situation *(Baseline reality before they acted)*
- **T** - Task *(The specific objective/directive)*
- **O** - Obstacle *(The complexity/bottleneck faced)*
- **R** - Result *(The immediate corporate outcome)*
- **Y** - Yield *(The true, compounding internal capability gained from the ordeal)*

## Step 5: Mandatory Persist

Save the resulting markdown artifact directly into the local memory ledger.

**File:** `experience/YYYYMMDD-HHMMSS-milestone.md` (Generate a timestamped filename)

Report to the user:
