# /operate , Embed Tactical Rhythms

> *"The vector is set. Now we encode it into muscle memory."*

OPERATE bridges the abstract strategy into observable, trackable behavior. It forces the definition of non-negotiable rituals and kill criteria.

---

## Usage

```
/operate
```

## Step 1: Pre-Reasoning Governance (PASS) -> HUMAN APPROVAL REQUIRED

Before starting, restate the user's intent back to them using the PASS framework. Apply these permanent directives to their specific query:

- **Purpose**: Operationalize the abstract strategy vector into daily/weekly tactical rhythms.
- **Anti-Purpose**: Do NOT allow the user to list ambiguous goals without attached metrics or frequencies.
- **Success-Signal**: We extract specific execution rhythms; output is an Operations Manual via the RHYTHM framework.
- **Synthesis**: Transforms vector strategy into observable state changes (`operate/[rhythm].md`) that dictate behavior.

Present the calculated PASS table to the user and ask: *"Do you approve this execution intent?"*

## Step 3: Interrogate

Ask the user: *"To achieve your [Goal/Waypoint], what are the 3 non-negotiable daily/weekly rituals you are committing to? And critically, what is your Kill Criteria (when do you stop if it's not working)?"*

## Step 4: Synthesis Framework (RHYTHM)

Synthesize their answers into an `Operations Manual` using the **RHYTHM** framework:
- **R** - Ritual *(The non-negotiable daily/weekly action)*
- **H** - Habitat *(The environment or mental state it requires)*
- **Y** - Yield *(The specific expected output)*
- **T** - Tracking *(The lagging/leading metric)*
- **H** - Halt *(The explicit Kill Criteria—when to abandon the rhythm)*
- **M** - Momentum *(How it loops back to feed GROWTH)*
