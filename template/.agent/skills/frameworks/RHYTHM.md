# Skill: RHYTHM Framework

> *"The vector is set. Now we encode it into muscle memory."*

The **RHYTHM** framework converts abstract strategy into observable, trackable behavior. Its defining feature is the **Kill Criteria**: a pre-committed condition under which a rhythm is explicitly abandoned, preventing sunk-cost loyalty to broken habits.

---

## Framework Letters

| Letter | Dimension | Question It Answers |
|--------|-----------|---------------------|
| **R** | Ritual | The specific recurring action with trigger, frequency, duration, and context. |
| **H** | Habitat | Physical and cognitive conditions required. What must be true before the ritual can start? |
| **Y** | Yield | The concrete, named deliverable produced by each session. Not "progress." |
| **T** | Tracking | One leading metric (execution quality) + one lagging metric (outcome quality). |
| **H** | Hardline Kill | The specific measurable condition under which this rhythm is formally declared dead and replaced. |
| **M** | Momentum | How the ritual's output feeds back into the GROWTH vector for the next iteration. |

---

## The Calendar Test

Before persisting any RHYTHM output, apply the Calendar Test:
> "Could you paste this ritual into a calendar and know exactly what to do, when, and when to stop?"

If no: add specificity to R (trigger+frequency+duration) and H (Hardline Kill).

---

## Two-Metric Rule for Tracking

Every ritual requires both:
- **Leading metric**: Tells you the ritual is being executed correctly BEFORE you see results (e.g., "sessions completed this week: 3/3")
- **Lagging metric**: Tells you the ritual produced its intended outcome AFTER 4-8 weeks (e.g., "inbound opportunities from published content: N")

One metric only is insufficient for governance.
