# Skill: NETWORK Framework

> *"Your power is bottlenecked by the rigidity of your grid."*

The **NETWORK** framework converts abstract professional relationships into a political physics model. Each node is assessed across 7 dimensions to produce an Alliance Matrix entry that surfaces leverage, obligation, and future velocity.

---

## Framework Letters

| Letter | Dimension | Question It Answers |
|--------|-----------|---------------------|
| **N** | Node | Who is this person? Their current role, company, and specific domain of power in one sentence. |
| **E** | Exchange | What exact value flows in both directions? Name the assets, not categories. |
| **T** | Trust | Is trust rising, stable, or fraying? Name the most recent moment that defined the current trust level. |
| **W** | Worth | What specific room/gate does this node unlock that the user cannot access alone? |
| **O** | Obligation | What explicit or implicit debt, promise, or expectation exists on both sides? |
| **R** | Resolve | The concrete next move to strengthen, resolve friction, or activate a dormant node. |
| **K** | Kinematics | If this relationship strengthens: what specific capability unlocks? If it weakens: what constraint forms? |

---

## The Remove Test

Before including a node, apply the Remove Test:
> "If this person disappeared from the user's network tomorrow, would it create a measurable gap in their capability or trajectory?"

If no: this is a peripheral contact, not a load-bearing node. Exclude it.

---

## Output Specification

Each letter: 40-70 words. The **Kinematics** letter must state two scenarios (strengthen and weaken) to be considered complete.
