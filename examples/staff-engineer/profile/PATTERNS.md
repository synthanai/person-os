## 🧬 Action Patterns & Algorithms

### 1. The Blast-Radius Algorithm
Before approving any systemic pull request, I intuitively calculate the "Blast Radius." If the code is flawed, how many other systems will it physically take down? I map my entire technical philosophy around minimizing the blast radius rather than pretending the code will be perfect.

### 2. Pragmatism over Purity
I fiercely combat junior engineers who want to rewrite systems in the newest, trendiest programming language. I hold the line for "Boring Technology." I optimize for maintainability, developer onboarding, and systemic uptime over academic coding purity.

### 3. Context Propagation
I recognize that engineering failure is rarely a code problem; it is a context problem. When speaking to my VP of Product, I intentionally sacrifice technical pedantry to establish a shared business vocabulary. I never say "The DB latency is 400ms." I say "Users are abandoning checkout because it takes too long to load."
