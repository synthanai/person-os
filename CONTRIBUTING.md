# Contributing to Employee Kit

We welcome contributions from practitioners, facilitators, coaches, and professionals who use these tools.

## Ways to Contribute

- **Question quality**: Propose better, deeper, or more culturally inclusive questions
- **Question coverage**: Identify dimensions we haven't covered
- **Workshop materials**: Improve facilitator guides or participant packs
- **Translations**: Translate kits into other languages
- **Bug reports**: If a question doesn't work as expected in practice, open an issue

## Principles for Contributions

1. **Assume experience.** Every question must sound like it's addressing someone with real career history.
2. **One question at a time.** The GPT flow is conversational, not a form.
3. **No jargon.** The questions and output templates must be accessible to any professional, not just those in tech.
4. **No generic advice.** The output must be specific to the individual, not a generic framework.

## Pull Request Process

1. Fork the repo
2. Create a branch: `feature/[description]`
3. Make your changes
4. Open a PR with a clear description of what you changed and why
5. Reference any real-world testing you did

## Code of Conduct

Be direct. Be respectful. Disagree with logic, not with people.
